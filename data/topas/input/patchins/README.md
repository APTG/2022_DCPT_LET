# TOPAS Patch-In Snippets

These files are intended for `tools/patch_topas_testmode.py`, not for TOPAS
`includeFile` use.

The TOPAS input source of truth should be a self-contained file generated by:

```bash
dicomexportplan --test-mode --export-fmt topas --nozzle-side neg-z ...
```

That file keeps `Ge/gantryAngle`, `Ge/couchAngle`, the
`World/Couch/DCM_to_IEC/Gantry` frame, and `BeamPosition` in one TOPAS parameter
chain. The patcher removes dicomexport's test runtime block plus the test
`IsoBox/IsoScore` water scorer, then inserts snippets from this directory in its
place.

Suggested naming:

- `common_*.txt` for snippets shared by every plan/geometry, such as project
  physics settings, material definitions, scoring boxes, and native scorers.
- `geo*.txt` for geometry-specific slab phantoms and scenario output directory
  definitions.

Keep snippets self-contained and explicit. They are concatenated into the generated
`main.txt`; they should not contain `includeFile` lines.
